# Mobile-Responsive Audit — sparcd-tagger & sparcd-uploader

## Executive summary

**sparcd-tagger — not phone-usable today.** The core tag workspace and Browse shell are built on hard-coded pixel grids (`280px_1fr_340px`, `1fr_340px`, `320px_1fr`) that never collapse, so at 390px the image being tagged and the upload pane are crushed to slivers or pushed off-screen. On top of that, the most important actions (save, mark-questionable, next/prev, burst select, open the species loupe, open the cheatsheet) are keyboard- or hover-only with no touch affordance, and two write-path dialogs (Sync, Snapshots) are fixed-width and overflow every phone. This app needs both a responsive-layout pass and a touch-controls pass before it works on a phone.

**sparcd-uploader — degraded but closer.** Most steps live in single `max-w-2xl` columns that reflow, so there are no full blockers, but two surfaces fail hard: the FileList inspect step uses a ~558px six-column grid that overflows and a `d`-key-only file-removal path, and the folder picker (`webkitdirectory` / File System Access) does not work on iOS Safari, so an iPhone user cannot start an upload at all. The header overflows, the Settings guard modal is fixed at 440px, and form fields are sub-16px throughout (iOS auto-zoom).

**Top 5 cross-app themes**
1. **Fixed multi-column grids that never collapse** — `grid-cols-[…px…]` with no responsive prefix in Tag, Browse, FileList, RunMonitor; the dominant blocker class.
2. **Keyboard-only core actions** — save / questionable / next-prev / burst-select / file-removal / cheatsheet are bound to physical keys with no on-screen control.
3. **Hover-only affordances** — species loupe, key rebind/clear, info buttons use `opacity-0 group-hover:opacity-100`; invisible on touch.
4. **Sub-16px form inputs → iOS focus auto-zoom** — `text-[12px]/[13px]/[14px]` on nearly every input/select/textarea in both apps; no global guard.
5. **`100vh`/`vh` instead of dynamic viewport units** — shells and modal scroll caps pinned to `h-screen`/`vh`, so bottom toolbars and dialog footers hide under mobile browser chrome (plus fixed-width dialogs and no global `overflow-x` guard letting wide grids scroll the whole page sideways).

## Severity legend

| Severity | Meaning |
|---|---|
| **Blocker** | Core flow is unreachable or unusable on a 390px phone. |
| **High** | Major function broken or only reachable by keyboard/hover; serious degradation. |
| **Medium** | Usable but painful — overflow, scroll traps, sub-44px targets, iOS zoom. |
| **Low** | Polish — minor overflow, undersized non-critical targets, safe-area. |

Touch-target minimum referenced throughout is 44×44px; iOS auto-zoom threshold is a 16px input font.
